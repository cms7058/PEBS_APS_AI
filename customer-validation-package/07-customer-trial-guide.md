# AI 排产智能体客户试用操作手册

## 1. 试用目标

本次试用用于验证 AI 排产智能体是否能帮助计划员：

- 快速生成有限产能排产方案。
- 看清设备负荷、订单延期和瓶颈资源。
- 在插单、设备停机、缺料时快速评估影响。
- 导出排产计划、物料检查计划、物料配送计划。

## 2. 推荐试用流程

### 第一步：选择试用方式

如果暂时没有真实数据：

- 点击右侧智能体卡片中的“生成模拟数据”。
- 下载样例 CSV 查看格式。
- 点击“一键载入模拟数据”体验完整流程。

如果已有客户数据：

- 准备订单、工艺路线、设备资源三类文件。
- 按系统要求导入并确认字段智能匹配。
- 点击“应用导入”生成排产结果。

## 3. 客户需要准备哪些数据

### 必须准备

| 数据 | 关键字段 | 说明 |
| --- | --- | --- |
| 订单 | 订单号、零件号、数量、交期 | 告诉系统要生产什么、多少、何时交付 |
| 工艺路线 | 零件号、工序序号、工序名称、可选设备、标准工时 | 告诉系统每个零件怎么加工 |
| 设备资源 | 设备编号、设备名称、设备能力、日历编号 | 告诉系统哪些设备可以加工哪些工序 |

### 建议补充

| 数据 | 作用 |
| --- | --- |
| 班次日历 | 判断设备每天可用时间 |
| 物料齐套 | 判断缺料是否影响开工 |
| 在制品 | 判断当前任务已经做到哪一步 |
| 停机计划 | 判断维修、保养、故障对计划的影响 |
| 工装夹具 | 判断夹具、刀具、检具是否冲突 |

## 4. 页面主要功能

### AI 排产智能体

右侧对话框用于：

- 询问如何使用系统。
- 查询订单延期原因。
- 查询瓶颈设备。
- 生成计划员日报。
- 根据对话动态展示下一步动作卡片。

### 数据导入

客户文件表头不需要完全等于系统字段。系统会自动尝试匹配，例如：

- `订单编号` 可匹配到 `订单号`。
- `料号` 可匹配到 `零件号`。
- `需求数量` 可匹配到 `数量`。
- `交货日期` 可匹配到 `交期`。

如自动匹配不准确，计划员可手动选择对应字段。

### 排产甘特图

甘特图显示：

- 每台设备上的工序安排。
- 订单、零件、工序。
- 开始时间、结束时间、工序耗时。
- 延期工序和风险。

### 加工时序分组视图

支持三种查看方式：

- 按设备：查看每台设备将加工哪些订单和零件。
- 按订单：查看每个订单的工序流转。
- 按零件：查看每类零件的加工顺序。

### 三类场景验证

| 场景 | 验证问题 |
| --- | --- |
| 插单模拟 | 急单插入后，哪些订单会受影响 |
| 停机模拟 | 设备不可用后，任务能否转移或顺延 |
| 缺料模拟 | 物料未齐套时，订单如何延期和释放设备窗口 |

每个场景先展示输入变化、重排计算和观察点，客户确认后再执行重排。

## 5. 导出文件

系统支持导出：

- 排产计划表：包含按设备、按订单、按零件三个 sheet。
- 物料检查计划表：用于仓库或物控做齐套检查。
- 物料配送计划表：用于现场按设备加工顺序送料。
- 试用报告：用于内部复盘和 POC 汇报。

## 6. 试用时重点观察

请客户重点判断：

- 排产结果是否符合计划员基本认知。
- 设备上的加工顺序是否可解释。
- 延期原因是否合理。
- 插单、停机、缺料后的变化是否清晰。
- 导出的表格是否能用于内部沟通。
- 如果接入真实 BOM、班次、在制品，是否愿意进一步验证。

## 7. 试用限制说明

当前版本是 MVP 试用版：

- 用于验证业务价值，不作为正式生产计划依据。
- 物料需求为试用版规则生成，正式版本需接入客户 BOM。
- 试用次数和试用周期有限制。
- 正式上线需要后端账号、权限、租户、数据接口和算法服务。

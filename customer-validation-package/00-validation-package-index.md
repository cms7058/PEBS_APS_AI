# AI 排产智能体客户验证包总览

## 使用目的

本验证包用于在 MVP 阶段快速判断客户是否有真实排产痛点、是否具备数据条件、是否愿意进入付费 POC 或试点。

适用对象：

- CNC 机加工企业。
- 汽车零部件供应商。
- 多品种、小批量、交期压力大的离散制造企业。
- 当前主要依赖 Excel、ERP/MES 导出表和计划员经验排产的工厂。

## 推荐验证路径

### 路径 A：客户暂时没有数据

1. 使用 MVP 页面中的“生成模拟数据”。
2. 演示基础排产、甘特图、加工时序、订单风险。
3. 演示插单、停机、缺料三个场景。
4. 导出排产计划表、物料检查计划表、物料配送计划表。
5. 让客户确认这些输出是否贴近真实计划工作。

目标：验证客户是否理解价值，是否愿意提供真实数据。

### 路径 B：客户愿意提供脱敏数据

1. 发给客户 `AI_APS_客户数据模板.xlsx`。
2. 客户至少填写订单、工艺路线、设备资源三张表。
3. 导入 MVP，检查字段智能匹配。
4. 生成基础排产方案。
5. 做插单、停机、缺料场景验证。
6. 复盘结果并填写成功标准评分表。

目标：判断是否进入付费 POC。

## 最小数据要求

MVP 试排最少需要三类数据：

- 订单：订单号、零件号、数量、交期。
- 工艺路线：零件号、工序序号、工序名称、可选设备、标准工时。
- 设备资源：设备编号、设备名称、日历编号、设备能力。

建议补充：

- 班次日历。
- 物料齐套状态。
- 在制品状态。
- 设备停机计划。
- 工装夹具约束。

## 客户验证会议建议

### 第一次会议：痛点和数据确认

目标：

- 了解当前排产流程。
- 确认是否存在插单、停机、缺料、延期痛点。
- 确认客户能否提供脱敏数据。

输出：

- 是否进入数据试排。
- 明确客户侧负责人。
- 明确数据提交时间。

### 第二次会议：MVP 试排演示

目标：

- 展示基础排产结果。
- 展示插单、停机、缺料场景。
- 让计划员判断结果是否有参考价值。

输出：

- 成功标准评分。
- 关键差距清单。
- 是否进入 POC。

### 第三次会议：POC 商务推进

目标：

- 明确 POC 范围、周期、费用。
- 明确客户数据接口和项目边界。
- 明确成功后试点或采购路径。

输出：

- POC 方案。
- 项目排期。
- 商务报价。

## Go / No-Go 判断

建议进入 POC 的信号：

- 客户愿意提供真实或脱敏数据。
- 计划员认为排产结果可讨论、可修正、可参考。
- 插单、停机、缺料场景击中真实痛点。
- 生产负责人愿意参与复盘。
- 客户愿意讨论付费试点。

暂缓推进的信号：

- 客户只看概念，不愿提供数据。
- 没有计划员参与。
- 当前排产问题不痛，或已有 APS 运行良好。
- 数据完全不可导出，且客户无意改善。
- 只要求免费长期试用。

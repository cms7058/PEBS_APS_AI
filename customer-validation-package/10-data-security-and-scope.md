# 数据脱敏与试用边界说明

## 1. 数据脱敏建议

客户在 MVP 或 POC 阶段可以先提供脱敏数据，不需要提供敏感商业信息。

建议脱敏方式：

- 客户名称可替换为客户 A、客户 B。
- 订单号可保留格式但替换流水号。
- 零件号可保留结构但替换关键编码。
- 单价、金额、成本信息无需提供。
- 工艺参数可只保留排产所需字段。
- 设备名称可替换为 CNC-01、CNC-02 等编号。

## 2. MVP 最小数据范围

最少需要：

- 订单。
- 工艺路线。
- 设备资源。

如果缺少以上任意一类数据，系统只能演示模拟数据，不能判断客户真实排产效果。

## 3. 不建议在 MVP 阶段提供的数据

除非客户明确同意，MVP 阶段不建议提供：

- 产品报价。
- 客户合同。
- 财务成本。
- 员工个人信息。
- 供应商敏感价格。
- 图纸和三维模型。
- 涉密军工或受监管产品数据。

## 4. 试用边界

当前 MVP 用于验证：

- 排产流程是否可闭环。
- 客户数据能否导入。
- 计划员是否看得懂结果。
- 异常场景是否有价值。
- 客户是否愿意进入付费 POC。

当前 MVP 不承诺：

- 直接替代正式 APS。
- 自动下发生产计划。
- 与 ERP/MES 实时集成。
- 覆盖客户全部现场约束。
- 产出可直接执行的最终生产命令。

## 5. 正式 POC 前需确认

进入付费 POC 前，双方应确认：

- 数据范围。
- 数据脱敏方式。
- 使用人员。
- 验证周期。
- 输出物。
- 成功标准。
- 是否允许项目方保存样例数据用于模型调试。

## 6. 客户侧建议参与人员

- 生产计划员。
- 生产经理或车间主任。
- 工艺工程师。
- 物控或采购负责人。
- IT/MES/ERP 负责人。
- 运营负责人或总经理。

# 客户访谈提纲

## 访谈目标

判断客户是否存在强排产痛点，是否具备 POC 数据条件，是否有明确试点意愿和预算可能。

## 访谈对象

- 生产计划员。
- 车间主任。
- 生产经理。
- 物控/采购负责人。
- 工艺工程师。
- IT/MES/ERP 负责人。
- 总经理或运营负责人。

## 基础情况

1. 公司主要产品是什么？
2. 属于 CNC 机加工、汽车零部件，还是其他离散制造？
3. 年产值大概是多少？
4. 有多少个车间、产线、设备？
5. 订单模式是按单生产、预测生产，还是两者混合？
6. 典型订单交期是几天到几周？

## 当前排产方式

1. 现在用什么工具排产？Excel、ERP、MES、APS，还是人工白板？
2. 谁负责排产？
3. 每天或每周排产要花多长时间？
4. 排产频率是多少？每天、每班、每周，还是随时调整？
5. 排产计划会下发到哪里？MES、纸质工单、微信群、看板？
6. 计划变更如何通知车间？

## 核心痛点

1. 最常见的排产问题是什么？
2. 插单频率高吗？插单后通常如何处理？
3. 缺料会不会经常影响计划？
4. 设备故障后重排要多久？
5. 哪些设备或工序最容易成为瓶颈？
6. 订单延期主要原因是什么？
7. 换型、调机、首件确认对产能影响大吗？
8. 排产结果和现场执行偏差大吗？

## 数据条件

1. ERP/MES 里是否有订单数据？
2. 工艺路线是否结构化？
3. 每道工序是否有标准工时？
4. 设备能力是否有清单？
5. 班次日历是否明确？
6. 当前在制品状态是否可导出？
7. 物料齐套状态是否可获取？
8. 历史实际开完工时间是否可获取？

## 决策与预算

1. 如果排产效率提升明显，谁会推动采购？
2. 是否有智能制造、MES、APS、数字化相关预算？
3. 可接受先做 POC 吗？
4. POC 成功后，是否愿意进入付费试点？
5. 采购更偏好 SaaS、私有化部署，还是项目制？

## POC 意愿判断

强意向信号：

- 愿意提供真实数据。
- 排产痛点由生产负责人主动提出。
- 明确存在延期、插单、瓶颈或缺料问题。
- 目前依赖 Excel 和人工经验。
- 愿意安排计划员参与验证。

弱意向信号：

- 只想了解概念，不愿提供数据。
- 现有 APS 已运行良好。
- 工艺和订单数据完全不可导出。
- 没有业务负责人参与。

## 访谈结束时要争取的结果

- 获取一份脱敏订单数据。
- 获取一份工艺路线数据。
- 获取设备和班次清单。
- 约定 POC 范围。
- 明确客户侧负责人。
- 约定下一次排产结果复盘会议。
